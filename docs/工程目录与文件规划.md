## 6. java 目录

Java 侧负责网关、系统管理、知识治理流程编排、文档元数据、审核发布、检索测试、Chat 入口等能力。工程结构按阿里开发规范落地：包名全小写、类名 UpperCamelCase、方法和变量 lowerCamelCase、常量 UPPER_UNDERSCORE_CASE，禁止 Controller 写业务逻辑，禁止 Entity/DO 直接返回前端，禁止用 `Map` 代替明确对象。

### Java 总体规范

```text
根包名：com.aistudio

Controller：只做参数校验、鉴权入口、调用 Service、返回 Result。
Service：service 包下只放 I 开头接口，例如 IKnowledgeBaseService。
ServiceImpl：service/impl 包下放接口实现，例如 KnowledgeBaseServiceImpl。
Mapper：MyBatis-Plus Mapper 接口，接口名使用 `I` 前缀，例如 IKnowledgeBaseMapper，负责 SQL 映射，不写业务逻辑。
XML：SQL 写在 resources/mapper/**/*.xml，文件名不加 `I` 前缀，例如 KnowledgeBaseMapper.xml，不使用注解 SQL 作为主方式。
DO：数据库对象，放 entity 包，不返回前端。
Response：接口返回对象，和 Request 成对使用。
Query：数据库查询条件对象，由 Request 转换得到，不直接使用 Request 访问数据库。
Convert：对象转换类，优先使用 builder 风格构造对象。
Enum：状态、类型、阶段必须枚举化，不使用魔法值。
API 层对象：Request、Response 只作为接口入参和出参模型，不归类为 Entity/DO；Mapper 不接收 Request，不返回 Response。
注释：接口、实现类、核心方法必须写清楚职责；复杂业务逻辑、关键分支、状态流转、异常处理必须补充必要注释。
日志：Service 入口、异步任务投递、Python 回调、状态变更、审核发布、异常分支必须记录日志；日志要包含业务主键和关键状态，不记录敏感信息。
```

```text
java/
└── ai-knowledge-studio-server/
    ├── pom.xml
    ├── platform-foundation/
    ├── gateway-service/
    ├── system-service/

```

### platform-foundation 基础模块

Java 第一阶段创建 `platform-foundation` 基础模块，用于放三个服务都稳定依赖的基础能力。该模块不写具体业务逻辑，不访问业务数据库，不放 Controller、Mapper、Service。

```text

├── pom.xml
└── src/main/java/com/aistudio/foundation/
    ├── constant/
    ├── context/
```

`system-service`、`chat-service` 这类服务自己的 Web 异常处理、过滤器、业务配置仍然放在各自服务内部，不放进 `platform-foundation`。

### Mapper 和 XML 约束

数据库访问统一放在业务包下的 `mapper` 包中，Mapper 接口名使用 `I` 前缀，不再额外增加数据访问包装层或对应实现类。Mapper 只定义数据库访问方法，复杂 SQL 写入 XML；XML 文件名不加 `I` 前缀，但 `namespace` 必须指向 Mapper 接口全限定名。Mapper 入参使用 Query 对象，返回 DO、`IPage<DO>` 或基础类型，不接收 Request，不返回 Response。分页查询统一使用 MyBatis-Plus 的 `Page<T>` / `IPage<T>`。

```java
public interface IKnowledgeBaseMapper {

    KnowledgeBaseDO selectById(@Param("id") Long id);

    IPage<KnowledgeBaseDO> selectPage(Page<KnowledgeBaseDO> page, @Param("query") KnowledgeBasePageQuery query);
}
```

```xml
        from knowledge_base
        where id = #{id}
          and deleted = 0

    <select id="selectPage" resultType="com.aistudio.system.knowledge.entity.KnowledgeBaseDO">

        from knowledge_base
        where deleted = 0
        <if test="query.name != null and query.name != ''">
            and name like concat('%', #{query.name}, '%')
        </if>

        </if>
```

### Service 接口和实现约束

`service` 包下只放接口，接口名按用户约定使用 `I` 前缀；实现类必须放在 `service.impl` 包下，类名去掉 `I` 并以 `Impl` 结尾。ServiceImpl 依赖统一使用 `@Resource` 字段注入，不使用 `private final` 加构造方法注入。ServiceImpl 必须加 `@Slf4j`，关键业务入口和异常分支必须打日志。

```text
knowledge/service/IKnowledgeBaseService.java
knowledge/service/impl/KnowledgeBaseServiceImpl.java
```

```java
public interface IKnowledgeBaseService {

    Long createKnowledgeBase(KnowledgeBaseCreateRequest request);

    PageResult<KnowledgeBaseResponse> pageKnowledgeBase(KnowledgeBasePageRequest request);
}
```

```java
import jakarta.annotation.Resource;
import com.baomidou.mybatisplus.core.metadata.IPage;
import com.baomidou.mybatisplus.extension.plugins.pagination.Page;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Service;

/**
 * 知识库应用服务实现，负责知识库创建、查询和状态维护。
 */
@Slf4j
@Service
public class KnowledgeBaseServiceImpl implements IKnowledgeBaseService {

    @Resource
    private IKnowledgeBaseMapper knowledgeBaseMapper;

    @Resource
    private KnowledgeBaseConvert knowledgeBaseConvert;

    /**
     * 分页查询知识库列表。
     */
    @Override
    public PageResult<KnowledgeBaseResponse> pageKnowledgeBase(KnowledgeBasePageRequest request) {
        log.info("分页查询知识库列表开始，pageNo={}，pageSize={}，name={}，status={}",
                request.getPageNo(), request.getPageSize(), request.getName(), request.getStatus());

        // Request 只作为接口入参，进入数据库查询前必须转换为 Query。
        KnowledgeBasePageQuery query = knowledgeBaseConvert.toPageQuery(request);
        Page<KnowledgeBaseDO> page = Page.of(query.getPageNo(), query.getPageSize());
        IPage<KnowledgeBaseDO> knowledgeBaseDOPage = knowledgeBaseMapper.selectPage(page, query);

        // Mapper 返回持久化对象，返回前统一转换为接口 Response。
        List<KnowledgeBaseResponse> responseList = knowledgeBaseDOPage.getRecords().stream()
                .map(knowledgeBaseConvert::toResponse)
                .toList();
        log.info("分页查询知识库列表结束，total={}", knowledgeBaseDOPage.getTotal());
        return PageResult.of(responseList, knowledgeBaseDOPage.getTotal(), query.getPageNo(), query.getPageSize());
    }
}
```

### Convert 约束

Request、Query、DO、Response 之间的转换统一放在 `convert` 包，不在 Controller、Service、Mapper 里散写字段拷贝。对象转换优先使用 builder 风格。

```java
/**
 * 知识库对象转换器。
 */
public class KnowledgeBaseConvert {

    public KnowledgeBasePageQuery toPageQuery(KnowledgeBasePageRequest request) {
        return KnowledgeBasePageQuery.builder()
                .pageNo(request.getPageNo())
                .pageSize(request.getPageSize())
                .name(request.getName())
                .status(request.getStatus())
                .build();
    }

    public KnowledgeBaseResponse toResponse(KnowledgeBaseDO knowledgeBaseDO) {
        return KnowledgeBaseResponse.builder()
                .id(knowledgeBaseDO.getId())
                .name(knowledgeBaseDO.getName())
                .description(knowledgeBaseDO.getDescription())
                .status(knowledgeBaseDO.getStatus())
                .createTime(knowledgeBaseDO.getCreateTime())
                .updateTime(knowledgeBaseDO.getUpdateTime())
                .build();
    }
}
```
