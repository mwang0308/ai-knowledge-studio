# 企业级 RAG 知识库检索前准备工作人工需求说明

> 文档定位：人工需求分析文档  
> 适用范围：企业级知识库管理平台、RAG 文档入库前准备、审核发布、召回验证  
> 说明：本文只整理人工需求，不展开数据库设计、接口设计、模型选型、向量库实现、Embedding、Rerank 等技术实现内容。

---

## 1. 需求背景

企业级知识库不是简单上传文件后直接问答，而是需要先完成知识内容的组织、处理、检查和发布。

RAG 检索前准备阶段需要把企业内部 Word、PDF、Markdown、TXT、Excel、CSV 等文档，经过知识库归属、目录归类、上传校验、文档解析、内容结构识别、分片预览、召回测试、审核发布等流程，形成可检索、可追溯、可管理、可控制发布范围的知识内容。

---

## 2. 业务目标

系统需要支持管理员完成以下工作：

1. 创建和维护多个知识库。
2. 在知识库下维护目录结构。
3. 上传文档时选择所属知识库和所属目录。
4. 支持多种企业常见文档格式。
5. 对上传文件进行格式、大小、重复性校验。
6. 对文档进行解析，识别文本、标题、段落、表格、图片、页码、Sheet、行范围等内容。
7. 对大文件进行解析块分段处理。
8. 将多个解析块的局部结构合并成统一文档内容结构。
9. 根据文档内容结构生成最终知识分片。
10. 支持分片预览和分片详情查看。
11. 支持管理员配置分片规则。
12. 支持文档、目录、知识库范围的召回测试。
13. 支持重要文档轻量审核。
14. 支持审核通过后发布，发布后才参与正式问答检索。
15. 支持已发布文档下架、重新处理和重新发布。

---

## 3. 角色说明

| 角色 | 说明 |
|---|---|
| 知识库管理员 | 负责知识库、目录、文档、分片配置、发布状态等管理 |
| 目录负责人 | 负责某个目录下的文档维护、上传、检查 |
| 审核人员 | 对重要文档进行人工检查，执行审核通过或驳回 |
| 普通用户 | 只使用已发布知识进行问答或检索，不参与检索前准备流程 |

---

## 4. 整体管理层级

系统整体管理层级为：

```text
知识库
  └── 目录
        └── 文档
              └── 文档内容结构
                    └── 知识分片
```

示例：

```text
财务共享知识库
├── 报销制度
│   ├── 差旅报销制度.docx
│   └── 费用报销流程.pdf
├── 费用标准
│   ├── 差旅费用标准.xlsx
│   └── 住宿标准.csv
├── 审批流程
│   └── 报账审批流程说明.docx
└── 常见问题
    └── 报销 FAQ.md
```

文档不应直接无组织堆在知识库下，必须挂到具体目录中。

## 5. 知识库管理需求

### 5.1 知识库列表

知识库列表展示所有知识库，字段包括：

| 字段 | 说明 |
|---|---|
| 知识库名称 | 如财务共享知识库、人事制度知识库 |
| 知识库说明 | 说明知识库用途 |
| 目录数量 | 当前知识库下目录数量 |
| 文档数量 | 当前知识库下文档总数 |
| 已发布文档数 | 正式可检索的文档数量 |
| 待审核文档数 | 等待审核的文档数量 |
| 处理失败文档数 | 处理失败的文档数量 |
| 默认分片配置 | 当前知识库默认分片配置 |
| 状态 | 启用 / 停用 |
| 操作 | 进入、编辑、停用、删除 |

### 5.2 知识库详情

知识库详情页包含：

```text
知识库详情
  ├── 基本信息
  ├── 目录管理
  ├── 文档管理
  ├── 分片配置
  ├── 审核发布
  └── 召回测试
```

知识库详情需要展示：

- 文档总数。
- 已发布文档数。
- 待审核文档数。
- 处理失败文档数。
- 最近上传文档。
- 最近审核记录。
- 默认分片配置。
- 默认审核规则。

## 6. 目录管理需求

### 6.1 目录定位

目录用于组织知识库下的文档，是文档上传和管理的归属位置。目录表示“文档放在哪里”，不是文档分类，也不是标签。

### 6.2 目录功能

| 功能 | 说明 |
|---|---|
| 新增目录 | 新增一级、二级、三级目录 |
| 编辑目录 | 修改目录名称、说明、负责人 |
| 删除目录 | 只允许删除空目录 |
| 调整顺序 | 支持上移、下移或拖拽排序 |
| 移动目录 | 支持目录移动到其他父级 |
| 设置负责人 | 设置目录负责人 |
| 设置默认分片配置 | 该目录下文档默认使用的分片配置 |
| 设置默认审核规则 | 该目录下文档是否默认需要审核 |
| 查看目录文档 | 点击目录筛选右侧文档列表 |

### 6.3 目录删除限制

存在文档的目录不允许直接删除，应先移动或删除目录下文档。

---

## 7. 目录、分类、标签关系

| 概念 | 作用 | 示例 |
|---|---|---|
| 目录 | 表示文档放在哪里，是树形结构 | 报销制度、费用标准、审批流程 |
| 文档分类 | 表示文档是什么类型 | 制度文件、流程说明、费用标准、FAQ |
| 标签 | 补充检索和筛选标记 | 报销、差旅、审批、供应商 |

规则：

1. 一个文档必须属于某个知识库。
2. 一个文档必须属于某个目录。
3. 一个文档可以有一个分类。
4. 一个文档可以有多个标签。
5. 目录和分类不能混用。

## 8. 支持文档格式需求

| 文档类型 | 文件格式 | 是否支持 | 说明 |
|---|---|---|---|
| Word 文档 | `.doc`、`.docx` | 支持 | 制度、流程、手册、通知 |
| PDF 文档 | `.pdf` | 支持 | 制度文件、扫描件、盖章文件 |
| Markdown 文档 | `.md`、`.markdown` | 支持 | 技术文档、接口文档、说明文档 |
| 文本文档 | `.txt` | 支持 | FAQ、话术、规则说明 |
| Excel 文档 | `.xls`、`.xlsx` | 支持 | 标准表、清单、台账 |
| CSV 文档 | `.csv` | 支持 | 结构化表格数据 |
| PowerPoint 文档 | `.ppt`、`.pptx` | 可扩展 | 培训材料、宣贯材料 |

PDF 需要支持文本型 PDF 和扫描型 PDF。

Markdown 需要保留标题层级、列表、表格、代码块、引用内容、链接文本。

Excel / CSV 需要保留 Sheet、表头、数据行、合并单元格、表头与数据行关系。

## 9. 文件大小限制需求

| 文件类型 | 建议大小限制 |
|---|---:|
| Word：`.doc`、`.docx` | 50MB |
| PDF：`.pdf` | 100MB |
| Markdown：`.md`、`.markdown` | 20MB |
| TXT：`.txt` | 20MB |
| Excel：`.xls`、`.xlsx` | 50MB |
| CSV：`.csv` | 50MB |
| PPT：`.ppt`、`.pptx` | 100MB |

如果支持批量上传，建议限制：

| 限制项 | 建议值 |
|---|---:|
| 单次最多上传文件数 | 10 个 |
| 单次上传总大小 | 300MB |

超过限制时，系统应阻止上传并展示明确提示。

## 10. 文档上传需求

### 10.1 上传入口

文档上传应在某个知识库或目录下进行。

推荐入口：

```text
知识库详情 → 目录管理 / 文档管理 → 上传文档
```

### 10.2 上传字段

| 字段 | 说明 |
|---|---|
| 所属知识库 | 必填 |
| 所属目录 | 必填，文档必须挂到具体目录下 |
| 文档分类 | 制度文件、流程说明、费用标准、FAQ 等 |
| 文档标签 | 可选 |
| 分片配置 | 默认继承目录配置，可手动选择 |
| 是否需要审核 | 默认继承目录审核规则，可手动调整 |
| 审核人员 / 审核组 | 需要审核时选择 |
| 文件上传 | 支持多格式文档 |
| 备注说明 | 上传说明 |

### 10.3 上传校验

上传时应校验：

- 文件格式是否支持。
- 文件大小是否超过限制。
- 文件是否为空。
- 文件是否重复。
- 文件名是否合法。
- 是否选择所属知识库。
- 是否选择所属目录。
- 是否满足该目录上传规则。

文档管理支持两种视图：

| 视图 | 说明 |
|---|---|

| 目录维度文档列表 | 点击目录后，只查看该目录下文档 |

推荐布局：

```text
左侧：目录树
```

文档列表字段包括：

| 字段 | 说明 |
|---|---|

| 所属目录 | 文档归属目录 |
| 文件类型 | Word、PDF、Markdown、TXT、Excel、CSV |
| 文档分类 | 制度、流程、FAQ、标准表 |
| 标签 | 业务标签 |
| 处理状态 | 上传中、解析中、分片中、处理完成、失败 |
| 审核状态 | 待审核、审核通过、审核驳回 |
| 发布状态 | 未发布、已发布、已下架 |

## 12. 大文件处理需求

### 12.1 基本原则

大文件不应一次性完整解析全文后再拆分。系统应支持大文件分段处理能力。

要求：

- 原始文件保持完整，不拆成多个小文件。
- 拆分的是处理过程中的解析块 / 内容块。
- 解析块是中间处理单元，不等同于最终知识分片。
- 最终分片仍然面向检索和问答使用。

### 12.2 大文件处理流程

```text
原始文档
  ↓
判断是否为大文件
  ↓
生成多个解析块
  ↓
分别解析解析块
  ↓
识别局部标题、段落、表格、图片、页码等结构
  ↓
合并形成完整文档内容结构
  ↓
生成最终知识分片
```

### 12.3 解析块拆分依据

| 文件类型 | 解析块拆分依据 |
|---|---|
| PDF | 页码范围 |
| 扫描 PDF | 页码范围 |
| Word | 标题层级、段落范围、页码范围 |
| Markdown | 标题层级 |
| TXT | 段落范围、文本大小 |
| Excel | Sheet、行范围 |
| CSV | 行范围 |

---

## 13. 文档内容结构查看需求

文档详情页应提供统一的“文档内容结构”查看能力。  
不同文件类型的展示方式应不同。

| 文件类型 | 展示方式 |
|---|---|
| Word 有标题 | 章节结构树 |
| Word 无标题 | 段落组、内容单元、表格和图片结构 |
| Markdown | 标题结构树，附带代码块、表格、引用标识 |
| PDF 有目录 | 章节结构树 + 页码 |
| PDF 无目录 / 扫描 PDF | 页码范围 + 识别内容类型 |
| Excel | Sheet / 表头 / 数据行范围 |
| CSV | 表头 / 行范围 |
| TXT | 段落结构或弱标题结构 |

对于大文件，页面展示的文档内容结构，应由多个解析块的局部结构统一合并而成。

## 14. 解析块查看需求

解析块用于表示大文件处理过程中的中间处理单元。

解析块不作为分片预览主流程入口，而是作为文档处理详情中的高级信息。

推荐位置：

```text
文档详情
  ├── 处理进度
  ├── 文档内容结构
  ├── 分片预览
  ├── 召回测试
  └── 处理详情
        ├── 解析块记录
        ├── 处理日志
        └── 异常记录
```

解析块列表字段包括：

| 字段 | 说明 |
|---|---|
| 解析块序号 | 第几个解析块 |
| 解析块名称 | 如第 1 - 20 页、第三章报销流程 |
| 拆分方式 | 页码范围、章节范围、Sheet、行范围、段落范围 |
| 来源范围 | 页码、章节、Sheet、行号、段落范围 |
| 内容摘要 | 解析块简要内容 |
| 处理状态 | 待处理、处理中、成功、失败 |
| 生成分片数 | 该解析块生成多少分片 |
| 操作 | 查看详情、关联分片、重试处理 |

## 15. 分片配置需求

### 15.1 配置层级

分片配置支持多层继承：

```text
系统默认分片配置
  ↓
知识库默认分片配置
  ↓
目录默认分片配置
  ↓
文档单独分片配置
```

优先级为：

```text
文档单独配置 > 目录配置 > 知识库配置 > 系统默认配置
```

### 15.2 配置内容

| 配置项 | 说明 |
|---|---|
| 配置名称 | 分片配置名称 |
| 适用范围 | 系统 / 知识库 / 目录 / 文档 |
| 内容块配置 | 大文件解析块处理规则 |
| 分片策略 | 固定长度、按段落、按标题、混合策略 |
| 最大分片长度 | 单个分片最大内容大小 |
| 最小分片长度 | 避免过短分片 |
| 分片重叠长度 | 相邻分片保留上下文 |
| 文件类型规则 | Word、PDF、Markdown、TXT、Excel、CSV 不同规则 |
| 特殊内容处理 | 表格、图片、代码块、链接等处理方式 |
| 试切分预览 | 选择文档预览分片效果 |

配置修改后，默认只影响后续新上传文档。已有文档需要手动重新处理后才应用新配置。

## 16. 分片预览需求

分片预览用于展示文档最终生成的知识分片，是文档发布前检查的重要页面。

分片预览不依赖解析块作为前置入口。用户进入文档详情后，应可以直接查看文档内容结构、分片列表和分片详情。

推荐结构：

```text
分片预览
  ├── 左侧：文档内容结构
  ├── 中间：分片列表
  └── 右侧：分片详情
```

分片列表字段包括：

| 字段 | 说明 |
|---|---|
| 分片序号 | 当前分片顺序 |
| 分片 ID | 分片标识 |
| 来源章节 / 来源结构 | 来源章节、Sheet、段落组或页码范围 |
| 内容摘要 | 分片内容摘要 |
| 字符数 | 字符数量 |
| Token 数 | Token 数量 |
| 向量状态 | 未向量化、向量化中、已向量化、失败 |
| 操作 | 查看详情、复制内容、定位来源 |

## 17. 召回测试需求

原型中的“测试”按钮统一命名为：

```text
召回测试
```

召回测试用于验证文档分片和知识库检索效果。

测试范围包括：

| 测试范围 | 说明 |
|---|---|
| 当前文档 | 只测试一个文档 |
| 当前目录 | 测试目录下文档 |
| 当前知识库 | 测试整个知识库 |
| 已发布内容 | 模拟正式问答效果 |
| 未发布内容 | 管理端发布前验证 |

召回结果字段包括：

| 字段 | 说明 |
|---|---|
| 排名 | 命中排序 |
| 命中分片 | 分片 ID |
| 命中内容 | 分片内容摘要 |
| 来源文档 | 来源文件 |
| 来源目录 | 所属目录 |
| 来源章节 / 结构 | 来源章节、Sheet、页码等 |
| 相似度 | 检索相似度 |
| 操作 | 查看分片详情、定位来源 |

## 18. 轻量审核发布需求

对于重要文档，文档处理完成后不应直接进入正式知识库，而应先进入待审核状态。

审核规则支持层级继承：

```text
文档设置 > 目录设置 > 知识库设置
```

文档状态包括：

| 状态 | 含义 |
|---|---|
| 上传中 | 文件正在上传 |
| 解析中 | 文档正在解析 |
| 分片中 | 文档正在生成分片 |
| 向量化中 | 分片正在向量化 |
| 处理完成 | 文档处理完成，但尚未发布 |
| 待审核 | 等待审核人员检查 |
| 审核通过 | 审核人员确认可发布 |
| 审核驳回 | 审核不通过 |
| 已发布 | 文档进入正式知识库，可被检索和问答命中 |
| 已下架 | 文档已从正式知识库移除 |
| 处理失败 | 文档处理失败 |

审核操作包括：

- 审核通过。
- 审核驳回。
- 填写驳回原因。
- 发布。
- 下架。
- 重新上传。
- 重新处理。

---

## 19. 发布可见性规则

| 发布状态 | 管理端 | 正式问答 |
|---|---|---|
| 未发布 | 可见 | 不可命中 |
| 待审核 | 可见 | 不可命中 |
| 审核通过 | 可见 | 不可命中，等待发布 |
| 已发布 | 可见 | 可命中 |
| 已下架 | 可见 | 不可命中 |

## 20. 重新处理与版本需求

系统应支持文档重新处理。

适用场景：

- 修改分片配置后重新生成分片。
- 解析失败后重新处理。
- 审核驳回后重新处理。
- 文档处理结果不符合预期。

需求上应保留基本版本概念：

| 字段 | 说明 |
|---|---|
| 当前版本 | 当前有效文档版本 |
| 历史版本 | 历史上传或处理版本 |
| 处理记录 | 每次解析、分片、向量化记录 |
| 审核记录 | 每次审核通过或驳回记录 |
| 发布记录 | 每次发布或下架记录 |

---

## 21. 异常处理需求

| 异常 | 说明 |
|---|---|
| 文件格式不支持 | 上传了系统不支持的格式 |
| 文件过大 | 超过大小限制 |
| 重复文件 | 文件已存在 |
| 文件损坏 | 文档无法打开或解析 |
| 加密文件 | 文件需要密码 |
| 解析失败 | 文本、表格、图片解析失败 |
| 结构识别失败 | 无法识别标题、段落、表格等结构 |
| 分片失败 | 无法生成知识分片 |
| 向量化失败 | 分片无法完成向量化 |
| 召回测试无结果 | 测试问题未命中有效分片 |
| 审核驳回 | 审核人员认为不可发布 |

异常状态下应支持：

- 查看失败原因。
- 查看处理日志。
- 重新处理。
- 删除文档。
- 下载原文。
- 重新上传。

## 22. 总体业务流程

### 22.1 完整流程

```text
创建知识库
  ↓
维护目录
  ↓
设置目录默认分片配置和审核规则
  ↓
选择目录上传文档
  ↓
文件校验
  ↓
文档解析
  ↓
生成文档内容结构
  ↓
生成分片
  ↓
分片预览
  ↓
召回测试
  ↓
审核通过
  ↓
发布
  ↓
正式参与问答检索
```

### 22.2 驳回流程

```text
待审核
  ↓
审核驳回
  ↓
查看驳回原因
  ↓
重新上传 / 重新处理
  ↓
再次提交审核
```

### 22.3 下架流程

```text
已发布
  ↓
下架
  ↓
正式问答不可命中
  ↓
可重新发布
```

## 24. 需求结论

本需求围绕企业级 RAG 知识库检索前准备阶段展开，最终形成如下主线：

```text
知识库 → 目录 → 文档 → 文档内容结构 → 分片 → 召回测试 → 审核发布
```

其中：

- 知识库和目录用于组织知识。
- 文档上传必须绑定知识库和目录。
- 大文件通过解析块进行分批处理。
- 文档内容结构用于统一展示不同文件类型的解析结果。
- 分片预览用于检查最终知识分片质量。
- 召回测试用于验证检索效果。
- 审核发布用于控制正式问答可见性。
