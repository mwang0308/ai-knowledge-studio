# 模块边界

## Java 服务边界

### gateway-service

只处理入口层能力：

- 路由。
- 鉴权前置。
- traceId。
- 跨域。
- 限流。

不写业务逻辑，不访问业务数据库。

### system-service

负责后台系统和知识库治理：

- 用户、角色、菜单、权限。
- 知识库、目录、文档。
- 文档处理任务。
- 分片配置。
- 审核发布。
- 召回测试入口。
- Python 回调接收。

### chat-service

负责正式聊天：

- 会话。
- 消息。
- 知识问答入口。
- SSE 流式输出。

## Python 服务边界

- Java 管理状态和业务流程。
- Python 执行解析、分片、向量化、检索编排。
- Java 与 Python 通过 RabbitMQ 和 HTTP 回调解耦。
