# 数据库设计约定

## 通用字段

业务表必须包含：

- `id`
- `create_time`
- `update_time`
- `create_by`
- `update_by`
- `deleted`

## 命名约定

- 表名使用小写下划线。
- 字段名使用小写下划线。
- Java DO 使用 UpperCamelCase，以 `DO` 结尾。
- Mapper XML 文件名不加 `I` 前缀。

## 逻辑删除

重要业务表不做物理删除，统一使用 `deleted` 字段。

## 任务表

文档处理任务必须记录：

- 任务 ID。
- 文档 ID。
- 文档版本 ID。
- 当前阶段。
- 当前状态。
- 重试次数。
- 失败原因。
- 回调时间。

## 文档结构表

- `knowledge_document_structure` 保存解析后的目录树节点，使用 `section_uid`、`parent_section_uid` 表达树关系。
- `knowledge_document_parse_block` 保存按页码范围拆出的解析块，使用 `parse_block_uid` 作为解析块业务 ID。
- 目录结构、解析块、分片都必须同时保存文档业务 ID、版本业务 ID 和任务业务 ID，前端查询不得依赖数据库自增主键。
- Python 回调成功后，Java 必须从 `document_structure.json` 读取目录结构和解析块并落库，前端文档详情优先读取结构表，不允许只靠分片标题路径反推目录树。
