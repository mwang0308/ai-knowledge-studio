# PDF 目录层级与解析块展示修正

## 背景

用户反馈完整文档结构树层级展示不明显，PDF 解析出的目录层级不准确，并且点击目录树后解析块列表出现多个重复项。

## 差距判断

- 前端目录树只展示标题和分片数量，依赖默认缩进表达层级，缺少层级标识和页码提示。
- 前端解析块聚合优先使用旧 `blockIds`，当分片同时带有 `parseBlockId` 和历史块字段时，会把同一个页码解析块重复展示。
- Python PDF 标题层级推断过度依赖字号样式，未优先识别 `1`、`1.1`、`1.1.1` 等编号层级，导致目录树扁平或父子关系错误。

## 修改内容

- `DocumentView.vue` 目录树节点增加层级、页码展示字段，解析块聚合优先使用 `parseBlockId` 并按分片去重。
- `studio.css` 调整完整文档结构树宽度、层级标识、页码副文本和层级色条。
- `parse_chunk_pipeline.py` 调整 PDF 标题层级推断顺序：解析器显式层级优先，其次标题编号，最后使用文档标题和字号样式兜底。

## 验证

- `python -m compileall app`
- 合成 PDF 标题元素验证：文档标题、`1`、`1.1`、`1.1.1` 分别输出 1/2/3/4 级。
- `npm run build`
