# 文档解析

## 目标

将上传文档解析为结构化内容，形成可分片、可预览、可追踪的处理结果。

## 支持格式

- PDF
- Word
- Excel
- Markdown
- TXT
- CSV

## PDF 解析器配置

- 上传 PDF 时在分片配置中选择解析器，支持 `docling`、`mineru`、`pdf`。
- 默认解析器为 `docling`；`mineru` 适用于复杂版面或扫描件，`pdf` 为本地原生版面解析。
- 解析器类型必须保存到文档版本的分片配置快照，重新处理时继续使用该版本的解析器。
- 三种解析器必须统一产出完整目录结构、解析块和分片数据，不能只返回纯文本。

## 主流程

```text
文档上传
→ 创建处理任务
→ RabbitMQ 投递任务
→ Python 消费任务
→ 解析文档结构
→ 生成解析块
→ 生成分片
→ 写入 ES / Milvus
→ 回调 Java 更新状态
```

## 必须记录

- 文档 ID。
- 文档版本。
- 处理任务 ID。
- 当前阶段。
- 当前状态。
- 失败原因。
- 重试次数。
- 解析器名称。

## 异常处理

- 解析失败必须记录失败状态和错误信息。
- 支持人工重试。
- 重试时必须保留原任务记录和新任务记录之间的关系。
